# issue-4-llm-integration: LLM 呼び出しラッパー（分解）

目的:
- LLM 呼び出しを抽象化し、モデル切替が容易でタイムアウト/リトライを持つ安定したラッパーを提供する。

サブタスク:
- 02-1: LLM プロバイダ選定と認証方式確認（HuggingFace推奨）
- 02-2: 呼び出しラッパーの設計（timeout, retries, max_tokens 等）
- 02-3: プロンプトテンプレート管理（prompt templates）実装
- 02-4: モック呼び出し用のインターフェース（ユニットテスト用）作成
- 02-5: 単体テスト実装（成功/失敗/タイムアウトシナリオ）
