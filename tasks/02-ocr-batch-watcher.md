# 02 - OCR バッチ / フォルダ監視

目的
- 特定フォルダ（デフォルト: ~/Downloads/receipts/）を走査し、新規画像を OCR 処理する。運用はバッチ方式（cron など）または簡易 watcher（inotify）を選択可能にする。

受け入れ条件
- 指定フォルダ内の未処理画像を検出して処理する
- 処理中 / 成功 / 失敗をログに残す
- 重複処理を避ける（処理済みフラグ or 移動）

小タスク
- バッチ走査実装（既存 main.py のロジック再利用）
- ファイル移動（処理済み -> processed/）
- オプションで inotify ベースの watcher 実装（設定で切替）
