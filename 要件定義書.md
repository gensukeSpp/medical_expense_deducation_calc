## 1. 概要

確定申告における医療費控除を目的とした、領収書データ抽出および管理自動化アプリ。

## 2. 目的とスコープ

- **目的**: 医療費控除のためのデータ入力の簡略化。
- **ターゲット**: 一世帯（生計を共にする家族）の医療費管理。
- **スコープ**: Phase 1（領収書の読み込み、データ抽出、修正、キャッシュ管理）。

**【重要】ちなみに Phase 2 は、「計算」なので、現段階では含みません。**

## 3. 機能要件 (Phase 1)

1. **自動検知・抽出**: 特定フォルダへの画像投入をトリガーとした自動OCR解析。
2. **クリニック別テンプレート管理**: クリニックごとに抽出設定（座標補正等）をキャッシュし、次回以降の精度を向上させる。
3. **Human-in-the-Loop（フィードバック）**: ユーザーによる修正内容をシステムが学習（座標補正値の更新）し、以降の読み込みに反映させる。
4. **データ可読化**: OCR座標データだけでなく、ユーザー編集用の「ラベル(項目名)・値」形式へ変換・提示する。

## 4. システム構成と技術選定

- **言語**: Python
- **OCR層**: [PaddleOCR] (テキストと座標の抽出) 
- **判断層 (LLM)**:
    - **構造化LLM**: OCR結果から項目（日付、品名、金額）をJSON形式へ整形するために利用。
    - **命名抽出LLM**: OCRテキストから「クリニック名」を特定する専用モデル。
        - _推奨_: 日本語に強い軽量LLM（Llama 3, Mistral 7B 等）または、OpenAI API / Claude API 等。        
- **データ管理**: SQLite（クリニック別設定、座標キャッシュ、修正履歴の保存）。
- **UI/UX**: ユーザーが修正した値とOCR座標を紐付けるインターフェース。

## 5. ロジックフロー（「座標逆引き」モデル）

1. **解析**: OCRがテキストと座標を取得。
2. **特定**: 命名抽出LLMがクリニック名を特定し、DBから過去のテンプレート（座標補正値）を適用。
3. **提示**: 座標を隠し、人間が読みやすい「ラベル・値」リストとしてUIに提示。
4. **編集**: ユーザーが誤りを修正。
5. **フィードバック**: システムが修正前後を比較し、DB内の当該クリニックの座標オフセット（補正値）を更新（学習）。

## 6. 実装ステップ

- **ステップ 1**: PaddleOCRからテキストと座標を抽出し、JSONへ出力するエンジンの構築。
- **ステップ 2**: 抽出結果からキーワード（「合計」等）を頼りに金額を抜き出す文字列解析ロジックの実装。
- **ステップ 3**: クリニック名でテンプレートを管理するDBテーブルの設計と実装。
- **ステップ 4**: ユーザーによる修正を検知し、テンプレートの補正値を更新するフィードバックの実装。

### 「クリニック名の特定」に関する補足

OCRで拾ったテキストの集合から、クリニック名を正しく特定するために「別の LLM」を配置するのは非常に賢明な判断です。

- **なぜ必要か**: 領収書には多くの文字が混在するため、単純な正規表現では「〇〇薬品（調剤薬局）」や「〇〇クリニック」の判別が困難だからです。
- **推奨実装**: OCR結果の全テキストを抽出し、そのうち「冒頭の数行」を命名抽出LLMに渡し、「この領収書の病院名は？」と問いかけるシンプルなプロンプトで、高い精度でクリニック名を特定可能です。
